# Plan 4.1: EDD Foundation — Markdown Generator, Curated Filter & Deterministic Tests

## Objective
Build the markdown catalog generator and curated strategies filter ALONGSIDE their deterministic tests. Following EDD: define snapshot fixtures and test expectations first, then implement the generator that must satisfy them. All artifacts in this plan are deterministic (no LLM calls, no network, no GitHub Actions).

## Context
- .gsd/DECISIONS.md (ADR-004 — rich table format, relevance ≥ 0.5, per-category files)
- src/enrichment/schema.py (VALID_CATEGORIES, VALID_TAGS, 18-field output schema)
- src/pipeline.py (pipeline orchestrator — needs catalog generation step)
- src/config.py (PROJECT_ROOT, DATA_DIR)

## Tasks

<task type="auto">
  <name>Create markdown generator and curated filter</name>
  <files>
    src/generator/__init__.py
    src/generator/catalog.py
    src/generator/curated_filter.py
  </files>
  <action>
    Create a new `src/generator/` package with three files:

    **src/generator/__init__.py** — Empty init file.

    **src/generator/curated_filter.py** — Filters enriched items for curation:
    1. Define `CURATED_RELEVANCE_THRESHOLD = 0.5`
    2. Create `filter_curated(items: list[dict]) -> list[dict]`:
       - Returns items where `relevance_score >= 0.5` AND `enriched_at` is present (i.e., successfully enriched)
       - Items missing `relevance_score` or `enriched_at` are excluded
       - Returns a new list (does NOT mutate input)
    3. Create `save_curated_strategies(items: list[dict], path: Path = None) -> Path`:
       - Default path: `data/curated_strategies.json`
       - Filters using `filter_curated()`, then saves to JSON
       - Returns the path written to
    4. Create `load_curated_strategies(path: Path = None) -> list[dict]`:
       - Loads from `data/curated_strategies.json`
       - Returns empty list if file doesn't exist

    **src/generator/catalog.py** — Markdown catalog generator:
    1. Define `CATEGORY_FILES` dict mapping category to filename:
       ```python
       CATEGORY_FILES = {
           "battle-tested": "docs/battle-tested.md",
           "new-upcoming": "docs/new-upcoming.md",
           "experimental": "docs/experimental.md",
       }
       ```
    2. Create `sanitize_markdown(text: str) -> str`:
       - Escape pipe characters `|` → `\|`
       - Escape backticks in titles
       - Strip newlines from table cell content
       - Truncate summary to 150 chars for table cells
    3. Create `generate_category_table(items: list[dict], category: str) -> str`:
       - Filter items by `category` field
       - Sort by `relevance_score` descending
       - Generate a rich markdown table with columns:
         | Title | Source | Relevance | Effectiveness | Tags | Summary |
       - Title column should be a markdown link: `[title](url)`
       - Tags should be comma-separated
       - Summary truncated via `sanitize_markdown()`
       - If no items match the category, return a "No items yet" message
    4. Create `generate_category_file(items: list[dict], category: str) -> str`:
       - Returns full markdown file content for a single category
       - Header: `# {Category Title} Strategies`
       - Description paragraph explaining the category
       - The items table from `generate_category_table()`
       - Footer: `_Auto-generated by Agent Hub on {date}_`
    5. Create `generate_readme(items: list[dict]) -> str`:
       - Returns root README.md content
       - Header section: `# 🤖 Agent Hub — Agentic AI Strategy Catalog`
       - Tagline: brief description of the project
       - Summary stats: total items, per-category counts
       - For each category: section heading + top-10 items table + link to full docs file
       - Footer with auto-generation timestamp
    6. Create `write_catalog(items: list[dict], output_dir: Path = None) -> dict`:
       - Default output_dir: PROJECT_ROOT
       - Calls `generate_readme()` → writes to `{output_dir}/README.md`
       - Calls `generate_category_file()` for each category → writes to `{output_dir}/{category_file}`
       - Creates `docs/` directory if it doesn't exist
       - Returns stats dict: `{"readme_written": True, "category_files": 3, "total_items": N}`

    IMPORTANT design constraints:
    - Generator must produce IDENTICAL output for identical input (deterministic for snapshot testing)
    - Timestamps in footers should accept an optional `generated_at` parameter for test reproducibility
    - Empty categories must still produce a valid file (with "No items yet" message)
    - All markdown must be valid GitHub-Flavored Markdown
  </action>
  <verify>
    python -m pytest tests/test_catalog_generator.py -v --tb=short
  </verify>
  <done>
    - `filter_curated()` returns only enriched items with relevance >= 0.5
    - `generate_category_table()` produces a valid markdown table sorted by relevance
    - `generate_readme()` produces a README with stats, top-10 per category, links to category files
    - `write_catalog()` creates README.md + 3 category files
    - `sanitize_markdown()` escapes pipes, backticks, and truncates properly
  </done>
