  <name>Create enrichment processor</name>
  <files>
    src/enrichment/processor.py
  </files>
  <action>
    Create `src/enrichment/processor.py` — the core enrichment engine:

    **class EnrichmentProcessor:**

    1. `__init__(self, groq_client: GroqClient = None)`:
       - Accept optional injected GroqClient for testability
       - If None, create one from config (default behavior)
       - Store `self._groq` client reference
       - Initialize counters: `enriched_count`, `skipped_count`, `failed_count`

    2. `enrich_item(self, item: dict) -> dict`:
       - **Skip check**: If item already has `enriched_at` field, log skip and return item unchanged. Increment `skipped_count`.
       - **Build prompt**: Call `build_prompt(item)` from prompts.py
       - **Call LLM**: Use `self._groq.analyze_json(user_message=prompt, system_prompt=SYSTEM_PROMPT)`
       - **Validate**: Call `validate_enrichment(result)`. If invalid, log warnings and attempt to fix:
         - Clamp scores to 0-1 range
         - Normalize category to lowercase, strip whitespace
         - Filter tags to only VALID_TAGS
         - Re-validate. If still invalid, log error, increment `failed_count`, return item with `enrichment_error` field.
       - **Merge**: Call `merge_enrichment(item, result)` with model_used and enrichment_version
       - Increment `enriched_count`
       - Return enriched item

    3. `enrich_batch(self, items: list[dict], progress_callback=None) -> list[dict]`:
       - Iterate over items, calling `enrich_item()` for each
       - Call `progress_callback(i, len(items))` after each item if provided
       - On exception per item: log error, increment `failed_count`, continue to next item (never crash the batch)
       - Return list of all items (enriched or original on failure)
       - Log final summary: enriched/skipped/failed counts

    4. `get_stats(self) -> dict`:
       - Return `{"enriched": self.enriched_count, "skipped": self.skipped_count, "failed": self.failed_count}`

    IMPORTANT design constraints:
    - The processor must NEVER crash on a single item failure — always continue the batch
    - All LLM output must be validated before merge — never trust raw LLM output
    - The processor must be fully testable with a mocked GroqClient
    - Score clamping and tag filtering are "repair" steps — log them as warnings, don't fail
  </action>
  <verify>

  </verify>
  <done>
    - `enrich_item()` returns enriched dict with all 18 fields on success
    - `enrich_item()` skips already-enriched items (no LLM call)
    - `enrich_item()` handles LLM errors gracefully (no crash, sets enrichment_error)
    - `enrich_batch()` processes entire batch even if individual items fail
    - All 4 trajectory tests from Plan 3.1 now PASS (unskipped)
  </done>

  <action>
    **src/enrichment/eval_runner.py** — Golden dataset evaluation runner:

    1. `load_golden_dataset(path: str = "data/evals/golden_dataset.json") -> list[dict]`
    2. `score_enrichment(enriched_item: dict, expected: dict) -> dict`:
       - Compare enriched output against expected_enrichment
       - Return a score dict:
         - `category_correct` (bool): Does category match expected_category?
         - `relevance_in_range` (bool): Is relevance_score within expected range?
         - `effectiveness_in_range` (bool): Is effectiveness_score within expected range?
         - `tags_precision` (float): What fraction of returned tags are in expected_tags_must_include?
         - `tags_recall` (float): What fraction of expected_tags_must_include are in returned tags?
         - `tags_no_forbidden` (bool): None of expected_tags_must_not_include appear?
         - `summary_keywords_found` (float): Fraction of summary_must_mention found in summary
    3. `run_eval(processor: EnrichmentProcessor, dataset_path: str = None) -> dict`:
       - Load golden dataset
       - Run enrichment on each item (LIVE LLM calls)
       - Score each result
       - Compute aggregate metrics:
         - `category_accuracy` (float): fraction of correct categories
         - `relevance_accuracy` (float): fraction in-range
         - `effectiveness_accuracy` (float): fraction in-range
         - `avg_tag_precision` / `avg_tag_recall` (float)
         - `summary_keyword_coverage` (float)
       - Save results to `data/evals/eval_results_{timestamp}.json`
       - Return aggregate metrics dict
    4. `check_thresholds(metrics: dict) -> tuple[bool, list[str]]`:
       - Check against passing thresholds (from ADR-003):
         - category_accuracy >= 0.80
         - relevance_accuracy >= 0.85
         - all noise items have relevance_score < 0.3
         - avg_tag_recall >= 0.75
       - Return (all_pass, list_of_failures)
