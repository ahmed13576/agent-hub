# Plan 2.2: Pipeline Orchestrator, Dynamic Discovery & Entry Point

## Objective

<task type="auto">
  <name>Implement pipeline orchestrator</name>
  <files>
    - [src/pipeline.py](file:///c:/Users/moham/Documents/GSD/agent-hub/src/pipeline.py)
  </files>
  <action>
    Create `src/pipeline.py` with a `Pipeline` class:
    1. `__init__`: Instantiate all scrapers (GitHubScraper, RedditScraper, BlogScraper, TwitterScraper).
    2. `run() -> dict`: Execute all scrapers, collect results, de-duplicate, discover sources, save to DB. Returns stats dict.
    3. `_run_scrapers() -> list[dict]`: Run each scraper's `.scrape()`, catch exceptions per-scraper (log and continue), return aggregate list.
    4. `_deduplicate(items: list[dict]) -> list[dict]`: Remove items whose `id` (hash of url) already exists in the database. Also remove dupes within the current batch.
    5. `_discover_sources(items: list[dict])`: Extract URLs from `raw_content` and `description` fields using regex. Filter out blacklisted domains (github.com, reddit.com, youtube.com, twitter.com, wikipedia.org, medium.com, google.com, etc.). Track new domains in `data/discovered_sources.json` with {domain, first_seen, mention_count, sample_urls[]}.
    6. `_save_to_database(new_items: list[dict])`: Load existing `data/database.json`, append new items, write back.
    7. Log summary: X new items added, Y duplicates skipped, Z new domains discovered.
  </action>
  <verify>
    PowerShell: `python -m pytest tests/test_pipeline.py -v`
  </verify>
  <done>Pipeline orchestrator runs all scrapers, de-duplicates, discovers sources, and saves to database.</done>
</task>
