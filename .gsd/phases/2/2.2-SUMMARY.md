# Summary: Plan 2.2 — Pipeline Orchestrator & Entry Point

## What Was Done
1. **Implemented Pipeline orchestrator** (`src/pipeline.py`):
   - Instantiates and runs all 4 scrapers (GitHub, Reddit, Blog, Twitter).
   - Catches per-scraper exceptions — a single scraper failure doesn't crash the pipeline.
   - De-duplicates new items against both the existing database and within the current batch using SHA256 URL hashes.
   - Extracts URLs from `raw_content` and `description` using regex, filters against a comprehensive blacklist (30+ common domains), and tracks new domains in `data/discovered_sources.json` with mention counts and sample URLs.
   - Appends new items to `data/database.json`.
   - Returns detailed stats dict (new_items, duplicates_skipped, domains_discovered, total_in_db, errors).

2. **Implemented main entry point** (`src/main.py`):
   - Configures timestamped logging.
   - Validates config and prints warnings.
   - Runs the full pipeline and prints a summary.
   - Returns exit code 0 on success, 1 on fatal error.

3. **Created 13 pipeline tests** covering de-duplication (within batch + against DB), source discovery (extraction, blacklisting, mention counts), database save/load, full orchestration with mocked scrapers, error resilience, and URL regex correctness — all passing.
